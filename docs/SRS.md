## 1 引言

### 1.1 编制目的

随着社会经济发展，宠物消费市场规模持续扩大，宠物店作为产业链的核心环节，其运营效率与服务体验直接关系到商业成败。

本课题旨在设计并实现一个基于 Flask 的宠物识别与管理系统。该系统将深度融合深度学习图像识别技术与宠物店日常运营管理流程，通过宠物图像识别技术，辅助店员快速、准确地完成宠物品种识别，并自动创建或关联电子档案，解决人工录入效率低下、易出错的问题；将门店的核心业务（如洗美、寄养）流程数字化、标准化，实现全流程管理，然后收集与分析宠物档案及服务数据，生成多维度的统计报表，为门店的精准营销、库存管理和服务优化提供数据支持，提升运营效率与决策水平。

### 1.2 范围

1. 本软件产品为：**宠物识别与管理系统**。
2. 产品旨在：为中小型宠物店提供一体化的数字化运营解决方案。其主要功能包括：
   (1) 智能宠物识别：通过上传宠物图片，系统自动识别其品种，并关联或创建电子档案。
   (2) 全流程业务管理：实现洗美、寄养等服务的在线下单与记录归档。
   (3) 综合信息管理：集中管理宠物档案、服务项目与价格体系。
   (4) 数据可视化看板：提供经营关键指标（如服务量、品种热度）的图表化分析。
3. 产品利益、目的与最终目标：提升运营效率，通过自动化识别与数字化流程，大幅减少人工录入错误与重复劳动，缩短服务响应时间，改善客户体验，提供专业的品种识别服务、与完整的宠物信息档案，提升客户信任度与满意度，降低管理成本，减少对纸质记录的依赖，降低信息查找与管理的间接成本，并通过流程标准化减少运营差错，解决宠物店在宠物信息建档、服务过程跟踪、经营数据分析等方面存在的效率低下、信息孤岛与管理盲区问题，推动其向标准化、数字化、智能化运营转型。

## 2 总体描述

### 2.1 产品透视

本系统面向中小型宠物店，采用前后端分离的 B/S（Browser/Server）架构，前端使用 Bootstrap 5 构建响应式界面，后端基于 Flask 框架，数据库采用 SQLite，配合 PyTorch 深度学习模型实现宠物品种自动识别功能。

### 2.2 用户特征

| 用户角色 | 描述 | 是否需要登录 |
|---------|------|------------|
| **顾客/访客（Guest）** | 普通消费者，浏览商城、提交领养申请，不需要账号 | 否 |
| **店员（Staff）** | 系统日常使用者，负责宠物档案录入、服务工单处理、领养审核、订单管理 | 是 |
| **管理员（Admin）** | 系统维护者，拥有全部操作权限，可管理用户账号、系统配置 | 是 |

### 2.3 约束

- **技术约束**：采用 Flask + SQLite 技术栈，PyTorch 模型需运行在支持 CUDA 或 CPU 的环境中。
- **部署约束**：系统面向单机或局域网部署，不支持分布式多节点部署。
- **数据约束**：所有数据存储在本地 SQLite 数据库文件中。

### 2.4 假设与依赖

| 编号 | 假设/依赖 | 描述 |
|------|-----------|------|
| ASSUM-001 | 用户具备基本计算机操作能力 | 店员可熟练使用浏览器操作系统 |
| ASSUM-002 | 运行环境具备 Python 3.8+ | 系统依赖 Python 运行环境 |
| ASSUM-003 | breeds.txt 品种库文件存在 | AI 识别模块依赖品种名称映射文件 |
| DEP-001 | PyTorch 模型文件可用 | BreedClassifier 依赖预训练模型文件 |
| DEP-002 | 上传目录可写 | 宠物图片上传需要可写的文件系统 |
